Working Method

Every phase must follow the same disciplined workflow.

Step 1

Inspect the existing codebase.

Determine:

what already exists
what is reusable
what should remain untouched

Reuse existing architecture wherever possible.

Avoid duplication.

Step 2

Produce a scientific design audit.

Before implementing anything:

identify dependencies
identify risks
identify reusable infrastructure
identify missing components

No implementation before the audit.

Step 3

Implement only the approved scope.

Changes must be:

additive
deterministic
versioned
reproducible
append-only

Never modify historical MarketStates.

Never invalidate previous discoveries.

Step 4

Validate the implementation.

Produce engineering evidence including:

unit tests
causality tests
symmetry tests
determinism tests
compatibility tests
governance verification

Do not claim success without evidence.

Step 5

Only after implementation passes validation should the next scientific phase begin.
